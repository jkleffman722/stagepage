# Stage Plot — Research Reference

> **Purpose:** Define every element a stage plot contains so StagePage can build a structured visual builder that replaces the current PDF/design-tool workflow.

---

## What a Stage Plot Is

A stage plot is a bird's-eye view diagram of a touring act's complete live setup on stage. It is the **spatial/visual counterpart to the input list** — the plot shows WHERE everything goes, the input list shows HOW to connect it.

A clear stage plot can save 30+ minutes of load-in and reduces show-day surprises. An outdated or inaccurate stage plot is worse than none at all — the crew will trust it and set up the wrong show.

---

## What It Communicates

- Exact positions of all performers, instruments, and gear
- Microphone and stand types at each position
- Monitor wedge positions and mix assignments
- Power drop locations
- Stage dimensions and minimum size requirements
- Equipment responsibility: what the tour brings vs. what the venue provides
- Cable and snake routing

## Stage Orientation (Universal Convention)

**Always drawn from the audience's perspective looking at the stage.**

- Stage Left (SL) = performer's left = RIGHT side of diagram
- Stage Right (SR) = performer's right = LEFT side of diagram
- Downstage (DS) = front of stage, closest to audience
- Upstage (US) = back of stage

**Violating this convention is one of the most common and costly mistakes.** Every crew member expects this orientation — a reversed plot causes setup confusion at load-in.

## Elements That Appear on a Stage Plot

### Performers & Instruments
- Each band member by name and instrument
- Position labeled: DSL, DSC, DSR, USL, USC, USR
- For multi-guitarist bands: "GTR 1 (SR)" and "GTR 2 (SL)" — never just "Guitar"

### Microphones & Stands
- Position of every mic on stage
- Stand type for each: boom w/ round base, boom w/ tripod, straight stand, clip, headset, gooseneck
- Wired vs. wireless indication
- Channel number labeled near each mic (links to input list)

### Amplifiers
- All guitar and bass amps, labeled and positioned
- Quantity and placement (e.g., "2× Marshall 4×12 SR")
- Each amp linked to its input list channel

### DI Boxes
- Location and quantity
- Active vs. passive
- Which instruments feed each DI

### Monitor System
- **Wedge positions:** Rectangle/trapezoid shape pointing toward performer
- Label each wedge with its mix number: "Mix 1," "Mix 2"
- **IEM:** Mark receiver/antenna positions on stage
- Monitor mix assignments: which performer hears which mix

### Power Drops
- Lightning bolt symbol at each AC outlet location
- Quantity of outlets at each location
- Helps crew run power before load-in

### Stage Infrastructure
- Drum riser: dimensions + height (e.g., "8×10 ft platform, 12 in. high")
- Carpet/stage mat placement and dimensions
- Backdrops, scrims, or risers for other performers
- Truss/lighting positions if relevant

### Cable Infrastructure
- Snake cable run from stage box to FOH
- Snake format: "24×8 snake required"
- Sub-snake positions (e.g., drum sub-snake, keys sub-snake)
- Cable routing paths

### Wireless Systems
- Wireless mic positions and frequency/channel numbers
- IEM transmitter locations and antenna positions
- RF coordination notes for multi-act events

### Legend (Required)
- **Filled shapes** = Tour provides
- **Outline/hollow shapes** = Venue provides
- Any color coding used

## How Stage Plot Connects to Input List

These are two halves of one advance package. They must stay in sync:

| Stage Plot | Input List |
|---|---|
| Shows WHERE mic is positioned | Shows WHAT channel it connects to |
| Shows monitor wedge placement | Shows what instruments are in each mix |
| Visual / spatial reference | Technical / electrical reference |
| Tells crew where to set up | Tells crew how to patch |

**Every mic on the stage plot should have its channel number labeled.** Channel numbers must match exactly between documents. When one is updated, the other must be updated too. Both carry the same version number and date.

## Stage Plot by Ensemble Size

### Solo Act / Singer-Songwriter
- 1–2 vocal mics, 1 instrument, minimal monitoring
- Single-page, simple layout
- 2–4 inputs, 1–2 monitor sends

### Small Band (4-piece)
- Drum kit, bass amp, 1–2 guitar amps, 2–4 vocals
- Clear amp labeling (GTR 1, GTR 2, BASS)
- 3–4 monitor wedges with mix assignments
- 12–18 inputs total

### Full Production (8+ members)
- Multi-page document with detail views
- Drum riser with dimensions
- Brass section, multiple keys stations, multiple vocal positions
- Wireless IEM antenna positions
- 6–10+ monitor mixes
- May include side elevation for truss/rigging heights
- 24–48+ inputs

## Critical Rules (From Industry)

1. **Always include minimum stage dimensions in the header** — "Minimum stage: 24 ft wide × 18 ft deep"
2. **Include a scale indicator** — "1 inch = 2 feet" or dimension arrows between items
3. **Never put channel numbers, DMX addresses, or hospitality info on the stage plot** — those belong elsewhere
4. **Always include a legend** — filled = tour provides, outline = venue provides
5. **Font size must be legible when printed** — minimum 12pt for labels
6. **Filename convention:** `ArtistName_StagePlot_v2.3_2024.pdf`

## Most Common Mistakes

| Mistake | Impact |
|---|---|
| No version number/date | Venue uses outdated plot, sets up wrong show |
| Stage orientation reversed | Equipment positioned backwards |
| No minimum stage dimensions | Band doesn't fit on stage |
| No scale/dimensions | Crew misjudges spacing and placement |
| No legend (who provides what) | Missing or duplicated equipment at load-in |
| Monitor mixes not specified | Monitor engineer can't set up mixes; soundcheck chaos |
| Channel numbers missing | Plot and input list can't be cross-referenced |
| Wireless frequencies not shown | RF interference during performance |
| Mic stand type not noted | Wrong stand creates safety hazard or performance issue |
| Plot doesn't match input list | Inconsistency discovered during patching |
| Sent too late | Venue can't prepare or source missing equipment |

## Current Tools & Their Gaps

| Tool | Strengths | Gaps |
|---|---|---|
| Tecrider.com | Free, 200+ icons, PDF export | Can't rotate objects (free tier); no input list integration |
| Stage Plot Pro | Clean UI, free in early access | Uncertain pricing; limited customization |
| StageBuilder Pro | 3D visualization, professional | Expensive; overkill for most acts |
| Canva / Illustrator | Full design control | No audio-specific symbols; no integration with input list |
| PDF + Excel (most common) | Maximum flexibility | Completely manual; no cross-referencing; no validation |

**The universal gap:** No tool integrates the stage plot with the input list, provides version control with approval workflows, or generates a complete advance package.

---

## Delivery & Format

- **Format:** PDF (primary), high-res PNG (secondary)
- **Print-friendly:** Must work in black and white
- **Copies at load-in:** Bring 3–5 printed copies for crew chief, sound engineer, stage manager
- **Distribution:** Venue production, FOH engineer, monitor engineer, stage manager, tour manager
- **Timeline:** Send with initial advance package (2–4 weeks out); resend final version 1 week out; bring printed copies to load-in
